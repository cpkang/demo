            # Vue项目目录中主要文件夹介绍

            # 一、build———webpack配置目录

            build文件夹里面是对webpack开发和打包的相关设置，包括入口文件、输出文件、使用的模块等

            # 二、 mock———生成随机数据，拦截ajax请求

            当前端工程师需要独立于后端并行开发时，前后端会约定一份接口文档，但前后端开发进度并不一致，当后端没有完善接口功能时，前端需要在本地模拟数据返回。（mockjs使用场景，通常是后台开发接口未完成时，前端需要使用数据时候，就可以使用mockjs）。

            # 三、node_modules———npm加载的项目依赖模块

            node_modules是安装node后用来存放用包管理工具下载安装的包的文件夹。比如webpack等这些工具，通常执行npm install会生成。

            # 四、public———应急手段

            任何放置在public文件夹的静态资源都会被简单的复制，而不会经过webpack的处理。你需要通过绝对路径来引用它们。

            何时使用 public 文件夹？

            ​ 1.你需要在构建输出中指定一个文件的名字。

            ​ 2.你有上千个图片，需要动态引用它们的路径。

            ​ 3.有些库可能和webpack不兼容，这时你除了将其用一个独立的<script>标签引入没有 别的选择。

            # 五、src———源码目录（大部分应用程序代码所在的位置，开发的项目文件都在此文件中写）

            # 1.api

            在vue开发中，会涉及到很多接口的处理，当项目足够大时，就需要定义规范统一的接口。api文件夹中即放置了与后台api相关的文件。这里面有axios库的实例配置文件、使用配置的axios实例接入api获取数据的函数的集合的文件。

            # 2.assets

            放置静态资源，包括公共的css文件、js文件、iconfont字体文件、img图片文件以及其他资源类文件、之所以强调是公共的css文件，是因为要在组建的css标签里加入scoped标记，将其作用范围限制在此组件以及调用它的父级组件中，避免污染全局样式。

            # 3.common

            存放公共js方法。

            # 4.components

            存放的是开发需要的各种组件，各个组件联系在一起组成一个完整的项目。

            通常建议将可重复使用的视图放置在该文件夹中。诸如页眉，页脚，广告。网格或任何自定义控件之类的示例，如样式化的文本框或按钮。可以在视图内部访问一个或多个组件。

            注意：存放在这里的公共组件最好能应用于多个页面且为纯UI界面，不和接口数据有关。否则若对应某一个页面只需使用一次，应写入view对应的文件夹下。

            # 5.directive

            存放vue一些自定义指令。

            # 6.filter

            过滤器，存放一些过滤函数。

            # 7.icons

            存放icon的图标文件 .svg。

            # 8.layout

            页面公共内容布局。

            存放与页面布局相关的组件，例如导航栏、侧边栏。

            # 9.plugins

            包含所有Vue插件

            # 10.router

            放置路由设置文件，指定路由对应的组件。

            在Vue（通常是Vue Router）处理路由时，将定义路由以切换组件中使用的当前视图<router-view>。这些路线通常位于src/router中的文件里。

            # 11.store

            放置vuex需要的状态关联文件，设置公共的state、mutations等。

            # 12.styles

            存放外部css文件。

            # 13.utils

            存放统一管理工具函数。

            # 14.vendor

            第三方提供的工具，例如导出json数据到excel

            # 15.views

            用于存放各种页面组件（视图），如login等。一个视图可以具有一个或多个组件

            位于src/components下的组件不太可能在一条路由路线中使用，而位于src/views下的组件将被至少一条路由路线使用。

            # 16.App.vue

            项目主组件，也是项目所有组件和路由的出口，之后它会被渲染到项目根目录的 index.html 中显示出来。

            # 17.main.js

            项目入口文件，引入vue模块和App.vue组件以及路由router，需要在全局使用的一些东西也可以定义在这里.

            # 18.permission.js

            实现系统权限拦截，即判断用户是否登录；用于处理用户权限相关的逻辑。

            # 19.setting.js

            存放一些全局背景，例如是否显示侧边栏，是否固定页头...

            # 六、tests文件夹

            初始测试目录

            unit：单元测试，可以为每个组件编写单元测试
